### M3-1. `.claude/settings.json` `hooks.PostToolUse` 섹션 추가

design_final Sec.3.3 + Sec.5 (Q3 해소) 따라:
- `.py` 확장자 → `python3 -m py_compile <file>` (warning only, exit 0)
- `.sh` 확장자 → `shellcheck <file>` (warning only, exit 0)
- hook 스크립트는 **반드시 exit 0 반환** (차단 X, gradual adoption)

기존 settings.json 백업 후 `hooks.PostToolUse` 키만 추가 (다른 키 보존).

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "scripts/hook_post_tool_use.sh"
          }
        ]
      }
    ]
  }
}
```

### M3-2. `scripts/hook_post_tool_use.sh` 신규 작성 (warning only first)

```bash
#!/usr/bin/env bash
# 입력: PostToolUse JSON (stdin)
# 출력: stderr 경고만, exit 0 강제

file=$(jq -r '.tool_input.file_path // empty' 2>/dev/null)
[ -z "$file" ] && exit 0

case "$file" in
  *.py)
    python3 -m py_compile "$file" 2>&1 >&2 || true
    ;;
  *.sh)
    shellcheck "$file" 2>&1 >&2 || true
    ;;
esac

exit 0  # 항상 0 — 차단 X
```
