## 4. 작업 순서

```
S1. drafter (장그래) — 5개 항목 초안 작성 (병렬 가능, 순차도 무방, 오케 자율)
S2. reviewer (김민교) — 5개 초안 일괄 리뷰. 각 doc에 피드백 블록 추가
S3. drafter — reviewer 피드백 반영 → 수정 draft
S4. finalizer (안영이) — 최종본 확정 (피드백 블록 정리, AC 확정, 의존 그래프 수렴)
S5. 오케 (박지영) — planning_index.md 작성 (5개 합본 + 의존 그래프 + 작업 순서)
S6. 오케 — git_checkpoint.sh 로 commit (paths 명시, push 금지)
S7. 회의창에 📡 status 보고 (commit SHA + 산출물 6개 파일 경로)
```
