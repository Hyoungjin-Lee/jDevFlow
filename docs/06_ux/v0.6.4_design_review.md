## Sec. 1. Critical 발견 2건

### R-1 (Critical) — 19명 산식 명확화 + 박스 안 인원 정합

**drafter 본문:** Sec.4.3에서 "기획4+디자인4+개발7=15명 + 추가 3명 → 18명"으로 박았으나, 박스 안 추가 3명의 정체가 불명확합니다.

**Stage 5 design_final Sec.10.3 본문:** "박스 3개 (기획/디자인/개발) 18명 + 상단 Header status bar 1행에 PM 스티브 리 1명 = 19명. CTO·CEO 미표시."

**운영자 결정 본문 (Q1 = F-D3 final):** "F-D3 final 19명 표시 = 18명 박스 + PM 스티브 리 상단 status bar 1행 / CTO·CEO 미표시."

**모순 지점:**
- `operating_manual.md` Sec.1.2 조직도 = CEO + CTO + PM + 기획팀(4) + 디자인팀(4) + 개발팀(7) = **18명**
- 박스 3개 (기획/디자인/개발) = 4+4+7 = **15명**
- "박스 3개 18명"이라는 표현은 4팀 인원과 정합하지 않음 (15 ≠ 18)
- CTO·CEO 미표시 + PM = status bar 빼면, 박스에 표시 가능 인원 = 15명

**추정 의도 (reviewer 해석):**
- "19명 표시" = 박스 영역(15명) + PM status bar(1명) + CTO/CEO 미표시 placeholder 영역(2~3명?) — 표현 오류 가능성
- 또는 "18명 박스" = 운영자 결정 표기에서 "전체 조직 18명 중 박스 표시 영역" 의미 — 산식 모호

**R-1 정정 권고 (Critical, finalizer 본문 흡수 권장):**
- 옵션 A (정정 1) — "19명 표시 = 4팀 박스 안 15명 + PM status bar 1명 + (CTO·CEO 미표시 = 2명) + (HR팀 placeholder 미배정 = 1명) = 19명". 다만 미배정·미표시 영역을 산식에 포함하는 것은 표기 부자연스러움.
- 옵션 B (정정 2 — **권장**) — "19명 표시 = 박스 3개 안 15명 (기획4 + 디자인4 + 개발7) + PM 스티브 리 상단 status bar 1명 + (잠재 추가 슬롯 3개) = 최대 19명". CTO·CEO 미표시는 "보이지 않는 2명"으로 별도 명시.
- 옵션 C (정정 3) — 박스 안 4팀 인원만 정확히 명시 + "19명 표시"는 유보 → "16명 표시 (박스 15 + status bar 1) + CTO/CEO 미표시 2 = 조직 18 + 다중 bridge 영역" 단순화.

**finalizer 영역 결정 권장:** 옵션 B (잠재 추가 슬롯 3개 = HR팀 미배정 + 기타) 또는 옵션 C (단순화 16명) 중 운영자 의도 정합 옵션 채택. drafter 슬롯 인터페이스 영역에서는 본 산식 명확화만 박고 detail 매핑은 `personas_18.md` 의존(Stage 8 영역) 그대로 유지.

**Stage 8 진입 차단 영역:** Stage 8 구현 시 박스 layout 인원수가 명확해야 textual `Vertical` 컨테이너 행 수 산정이 정합되므로 Critical로 분류합니다.
