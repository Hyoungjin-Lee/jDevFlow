## 1. M1 — Monitor 인프라 검토 포인트

### 1.1 `scripts/monitor_bridge.sh` (신규)

**(MR1-1) signal injection 안전성**
- `📡 status <timestamp> <stage> <signal>` 출력에서 `$signal` 변수가 외부 입력(capture-pane 결과)을 그대로 echo하면 ANSI escape / control char 주입 가능.
- 검토: `printf '%s\n' "$signal"` 사용 vs `echo "$signal"` (BSD echo는 escape 해석). printf 채택 확인.

**(MR1-2) capture-pane 무한 루프 방지**
- `while true; do capture-pane ...; sleep N; done` 루프 사용 시 sleep 부재 / 0초면 CPU spike.
- 검토: `sleep` 호출 존재 + 주기 ≥ 1s. `trap` 핸들러로 SIGTERM 시 정상 종료 처리.

**(MR1-3) `-S -20` capture 범위 정합 (F-62-7c)**
- design_final Sec.5.3c 범위 `-S -20` 강제. 본 sketch는 `grep -E "capture-pane.*-S \-20" scripts/monitor_bridge.sh` ≥ 1 검증.

**(MR1-4) timestamp 함수 정합 (F-62-7a)**
- `date +%Y-%m-%d\ %H:%M:%S` 패턴. 공백 escape 처리 — quote 누락 시 인자 분리 위험.
- 검토: `date '+%Y-%m-%d %H:%M:%S'` (single quote) 또는 `date +%Y-%m-%d_%H:%M:%S` (underscore) 권장.

**(MR1-5) stage 명시 정합 (F-62-7b)**
- 신호에 "Stage X" 포함 검증. `grep "Stage" scripts/monitor_bridge.sh` ≥ 1.

**(MR1-6) 환경 변수 fallback (Q1)**
- `target_session="${TMUX_SESSION_BRIDGE:-bridge}"` 패턴 채택 검증. 미설정 시 기본값 fallback.

**(MR1-7) 파일 권한**
- `chmod +x scripts/monitor_bridge.sh` 적용 검증. `[ -x scripts/monitor_bridge.sh ]`.

**(MR1-8) shellcheck CLEAN**
- shellcheck 실행 → 경고 0건 또는 disable comment 명시 사유 1건 이내.

### 1.2 `scripts/update_handoff.sh` (개정 — Windows fallback)

**(MR1-9) BSD sed `-i ''` 인용 처리 (F-62-6)**
- macOS BSD `sed -i ''`는 빈 문자열 인자 필수. GNU `sed -i` (인자 없음).
- 검토: `if sed --version 2>/dev/null | grep -q GNU; then sed -i "..." else sed -i '' "..."; fi` 분기 정합.

**(MR1-10) globbing 배열 안전성 (R-6 정정 검증)**
- design_review Sec.2.3 (B2-1) 정정 영역: `[ -f "handoffs/active/HANDOFF_v*.md" ]` 글로빙 미작동 → 배열 안전 패턴 적용.
- 검토: `files=(handoffs/active/HANDOFF_v*.md); [ -f "${files[0]}" ]` 패턴 채택 확인.

**(MR1-11) readlink 호환성**
- BSD readlink (macOS)와 GNU readlink (Linux) 옵션 차이. 단순 `readlink HANDOFF.md` (옵션 없음)는 양 OS 호환.
- 검토: `readlink -f` 사용 시 macOS 미지원 → coreutils greadlink fallback 또는 옵션 미사용 권장.

### 1.3 `scripts/setup_tmux_layout.sh` (개정)

**(MR1-12) PostToolUse hook 발화점 추가 정합**
- design_final M1 → "Monitor 자동 재가동 hook 추가". 그러나 본인 M3 구현에서 `.claude/settings.json` `hooks.PostToolUse`는 `hook_post_tool_use.sh` (py_compile/shellcheck 경고용)로 점유.
- 검토: M1 Monitor 재가동 hook은 별도 hook 스크립트로 분리되었는지 (`scripts/setup_tmux_layout.sh` 내부 spawn 로직 또는 추가 hook entry 추가).
- 충돌 가능성: `hooks.PostToolUse` 단일 entry → 다중 hook 추가 시 array 확장 필요. 1.2 카더가든이 어떤 방식으로 통합했는지 commit 도달 시 확인.

## 2. M4 — 18명 페르소나 + 리뷰어 conditional 검토 포인트

### 2.1 spawn 자동화

**(MR4-1) `--dangerously-skip-permissions` 누락 검증 (F-62-9)**
- 모든 `claude` CLI 호출에 옵션 강제. `grep -E '^\s*claude\b' scripts/*.sh | grep -v 'dangerously-skip'` → 0건.
- 검토: spawn 스크립트(setup_tmux_layout.sh 등) 내 모든 claude 호출에 옵션 포함.

**(MR4-2) pane title 강제**
- tmux pane 1+3 모델에서 각 pane 식별 위해 `tmux select-pane -T "title"` 또는 `tmux send-keys` 시 prompt 명시 필요.
- 검토: spawn 시 pane title 또는 prompt 변경 명령 포함 여부.

**(MR4-3) sequential vs parallel spawn (Q4)**
- design_final Sec.6.4 Q4 답변: "tmux 1+3 모델 mock 시뮬레이션 후 동시성 이슈 발견 시 sequential fallback".
- 검토: spawn 로직이 동시 send-keys로 구성되어 있는지, 실패 시 sequential 분기 존재 여부.

**(MR4-4) 세션명 환경 변수 (Q1 연계)**
- `export TMUX_SESSION_ORC` 등 세션명 환경 변수 설정 정합.

### 2.2 conditional 분기

**(MR4-5) `stage9_review` 키 파싱 안전성 (B2-3 연계)**
- design_review Sec.2.3 (B2-3): `cut -d'"' -f4` 파싱은 JSON 들여쓰기에 따라 깨짐.
- 검토: `jq -r '.stage_assignments.stage9_review' .claude/settings.json` 사용 vs grep+cut. jq 사용이 안전.

**(MR4-6) jq 미설치 fallback**
- `command -v jq` 검사 + grep fallback 분기 존재 여부.
- 본인 `hook_post_tool_use.sh` 구현 시 적용한 패턴(jq 우선, grep fallback) 동일 패턴 권장.

**(MR4-7) 미정 값 처리**
- `stage9_review` 키 부재 또는 값이 "codex" / "claude" 외 임의 값일 때 동작.
- 검토: `case "$value" in "codex") ... ;; *) self-review fallback ;; esac` 분기 명시.

**(MR4-8) settings.json 키 보존**
- M4 변경이 `.claude/settings.json`에 새 키 추가 시 기존 키(`schema_version`, `stage_assignments`, `agents`, ...) 보존 검증.
- 본인 M3 구현에서 적용한 백업 패턴(`cp settings.json settings.json.bak.<date>`) 동일 적용 여부 검증.
