### 5.1 F-62-5 — 글로벌 충돌 우선순위 (프로젝트 > 글로벌)

**주제:** 프로젝트 `.claude/settings.json`과 글로벌 `~/.claude/CLAUDE.md` 설정 충돌 시 우선순위.

**결정:** **프로젝트 설정이 글로벌을 override. Specificity 원칙.**

**적용 메커니즘:**
- 프로젝트 `.claude/settings.json`에 정의된 모든 필드(workflow_mode, team_mode, stage_assignments, hooks, ...)는 글로벌 설정과 무관하게 최우선 적용.

- 프로젝트 설정에 키 부재 시만 글로벌 참조 (cascading lookup).

**검증 (AC-Global-1):**
- `grep "specificity\|프로젝트 > 글로벌" CLAUDE.md docs/operating_manual.md` → 0개 이상 hits (문서에 명시)
- `.claude/settings.json` 키가 존재하면, 이를 정확히 적용하는지 구현 단계에서 확인.

**Stage 8 구현:** CLAUDE.md Sec.1 또는 Sec.2에 명시. 예: "프로젝트 설정이 글로벌 설정을 override합니다(specificity 원칙)."

### 5.2 F-62-6 — Windows fallback sync (update_handoff.sh)

**주제:** Windows 환경에서 symlink 미지원 시 fallback 동기화 메커니즘.

**결정:** **BSD/GNU sed 분기 내부 포함. `update_handoff.sh` 확장.**

**구현 상세:**
```bash
# update_handoff.sh 내부: Windows fallback 분기
if [ -L "HANDOFF.md" ]; then
  # Unix/Linux/macOS: symlink 유지
  target=$(readlink HANDOFF.md)
  # ...
else
  # Windows fallback: copy + sync
  if [ -f "handoffs/active/HANDOFF_v*.md" ]; then
    cp -f handoffs/active/HANDOFF_v*.md HANDOFF.md
  fi
fi

# sed 호환성 분기 (GNU vs BSD)
if sed --version 2>/dev/null | grep -q GNU; then
  # GNU sed
  sed -i "s/status: .*/status: active/" "$file"
else
  # BSD sed (macOS)
```
