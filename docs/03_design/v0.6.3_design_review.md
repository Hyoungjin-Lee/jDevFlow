## Sec. 1. M1~M5 detail 5축 정합성 (검토 항목 1)

### 1.1 강점

**(S1-1) 5축 일관 적용 (Sec.2~Sec.10)**
모듈 경계(Sec.2), 데이터 흐름(Sec.3), 에러 경로(Sec.4), 테스트(Sec.7), 보안(Sec.8) 5축이 M1~M5 모두에 빠짐없이 적용됨. M별 인터페이스 컬럼(Sec.2.1)이 hook 발화점 / 설정 merge / 분기 로직으로 구체적.

**(S1-2) 의존 그래프 + critical path (Sec.2.2)**
ASCII 의존 그래프(M1→M3→M4→M5, M2 독립)가 plan_final Sec.1 마일스톤 의존 표와 정확히 일치. 5월 초 critical path(M1+M4) 인용도 일관.

**(S1-3) fail-closed vs fail-safe 원칙 분리 (Sec.4 말미)**
13건 에러 시나리오를 fail-closed(진행 차단 + 운영자 결정 안내)와 fail-safe(부분 실패 + 대체 경로 자동 진행)로 명시 분리. M1-EP3(capture-pane 범위 초과)이 fail-safe로 분류된 판단이 적절.

**(S1-4) 변경 대상 파일 신규/개정 단일화 표 (Sec.9)**
신규 3개 + 개정 8개 = 총 11개 파일이 M별 소유로 명시. plan_final F-2 정정(scripts/ 실증 후 단일화)이 본 draft에 일관 흡수됨. **본 리뷰 시점 실존 검증 결과 (Sec.5 상세):** `scripts/monitor_bridge.sh` / `docs/guides/ethos_checklist.md` / `~/.claude/CLAUDE.md` 모두 부재 → 신규 분류 정합.

**(S1-5) 보안 4축 (Sec.8)**
신호 위변조(M1) / settings.json 무단 편집(M4) / 글로벌 침투(M2) / Hooks 무한 루프(M3) — 잠재 위험을 4축으로 분리하고 각 완화책 제시. 특히 M3 무한 루프 방지 ("hook은 경고만, 파일 수정 금지")는 plan_final M3 결정과 정합.

**(S1-6) Stage 8 작업 분해 권장 순서 (Sec.11)**
의존 순서대로 M1 → M3 → M4 → M5 → M2 권장. M2 독립이지만 마지막 배치는 자원 분배상 합리적. AC 검증 카운트 명시(M별)도 양호.

### 1.2 보강 권고

**(B1-1) Sec.4 에러 경로 ↔ Sec.10 리스크 표 중복 영역 [낮음]**
Sec.4(M1-EP1~M5-EP2, 13건)와 Sec.10.1~10.5(M별 리스크 3~4건)가 BSD/GNU sed 호환성, capture-pane 범위, shellcheck 기존 script 실패 등에서 중복 표현. **권고:** finalizer 통합 시 "에러 경로 = 발화 위치 / 처리 정책 중심" vs "리스크 = 영향 영역 / 완화책 중심"로 컬럼 분리 명시 또는 통합 단일 표 검토.

**(B1-2) Sec.2.2 의존 그래프 — M2 표시 위치 [낮음]**

### 2.3 보강 권고

**(B2-1) Sec.5.2 F-62-6 sample 코드 — globbing 미작동 [중간]**
`l.281~284` `if [ -f "handoffs/active/HANDOFF_v*.md" ]; then cp -f handoffs/active/HANDOFF_v*.md HANDOFF.md; fi`에서 `[ -f ... ]` test 안의 글로브 와일드카드는 shell이 expand 하지 않으면 작동 안 함. **권고:**
```bash
files=(handoffs/active/HANDOFF_v*.md)
if [ -f "${files[0]}" ]; then
  cp -f "${files[0]}" HANDOFF.md
fi
```
형태로 정정. (R-6 정정 영역.)

**(B2-2) Sec.5.1 F-62-5 AC-Global-1 — "0개 이상 hits" 표현 모호 [낮음]**

**(B2-3) Sec.5.4 F-62-8 — `cut -d'"' -f4` 파싱 안전성 [낮음]**

**(B2-4) Sec.5.5 F-62-9 적용 범위 — `claude.*--dangerously-skip-permissions` 음성 검증의 false-positive [낮음]**
`l.406` `grep --color=never "claude" scripts/*.sh | grep -v "dangerously-skip"` 결과가 0이어야 한다는데, 주석/문서 내 "claude" 단어도 hit 가능. **권고:** `grep -E '^\s*claude\b' scripts/*.sh | grep -v 'dangerously-skip'` 형태로 명령 시작 위치 한정.
